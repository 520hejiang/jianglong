# 保姆级部署教程（零基础 · 全程免费）

> 跟着做，全程不需要付费。最终你会得到：一个能 24 小时自动写小说的后台（Cloudflare Worker）+ 一个网页控制台（Cloudflare Pages）。
>
> 全程大约 30–60 分钟。遇到任何报错，把**报错原文**发我即可。

整个过程分 6 大步：
1. 装好 3 个工具（Node、Git、Python）
2. 注册 2 个账号（Cloudflare、DeepSeek）
3. 把代码下载到电脑
4. 在 Cloudflare 上创建数据库等资源
5. 部署后台 + 控制台
6. 导入大纲，开始自动写

---

## 第 0 步：打开“终端”（命令行）

后面要敲命令，先学会打开终端：

- **Windows**：按 `Win` 键，搜索 **PowerShell**，点开它。黑色/蓝色窗口就是终端。
- **Mac**：按 `Command+空格`，搜索 **终端 / Terminal**，回车打开。

> 之后所有 `代码框里的命令`，都是**复制 → 粘贴到终端 → 回车**。Windows 在 PowerShell 里粘贴用鼠标右键。

---

## 第 1 步：安装 3 个工具

### 1.1 Node.js（运行部署工具用）
打开官网 https://nodejs.org/ ，下载 **LTS 版**（左边那个），一路“下一步”装完。

验证（终端里输入）：
```bash
node -v
```
能显示类似 `v20.x.x` 或 `v22.x.x` 就成功了。

### 1.2 Git（下载代码用）
官网 https://git-scm.com/downloads 下载安装（Windows 一路默认即可）。
验证：
```bash
git --version
```

### 1.3 Python（导入大纲用，3.8 以上）
- Windows：https://www.python.org/downloads/ 下载安装，**安装时务必勾选 “Add Python to PATH”**。
- Mac：通常自带，终端输入 `python3 --version` 能显示版本即可。

验证：
```bash
python3 --version    # Windows 若不行就试 python --version
```

✅ **检查点**：上面三条命令都能显示版本号，就继续。任何一条报“不是命令/not recognized”，说明该工具没装好或没加进 PATH，发我截图。

## 第 2 步：注册 2 个账号

### 2.1 Cloudflare（免费）
打开 https://dash.cloudflare.com/sign-up ，用邮箱注册、验证邮箱。**不用绑信用卡，不用买域名。**

### 2.2 DeepSeek（生成小说的大脑，按量付费、很便宜）
1. 打开 https://platform.deepseek.com/ ，注册登录。
2. 左侧找到 **API Keys（密钥）** → 创建一个 → **复制保存**（形如 `sk-xxxxxxxx`，只显示一次！）。
3. 充一点点钱（几块到几十块就够测试很久）。

> ⚠️ 这个 `sk-...` 密钥后面要用，先存到记事本里。**别发给任何人、别提交到 GitHub。**

## 第 3 步：把代码下载到电脑

在终端里依次执行（一行一行来）：

```bash
# 下载你的仓库（如果是私有库，git 会让你登录授权）
git clone https://github.com/520hejiang/jianglong.git

# 进入项目文件夹
cd jianglong

# 切换到我们开发的分支
git checkout claude/xianxia-novel-generator-i0essg

# 安装项目依赖（会下载一些东西，等它跑完）
npm install
```

✅ **检查点**：`npm install` 结束后没有红色 `ERR!` 大片报错即可（黄色 warning 不用管）。

---

## 第 4 步：登录 Cloudflare 并创建资源

### 4.1 登录
```bash
npx wrangler login
```
它会自动弹出浏览器 → 点 **Allow（允许）** → 回到终端看到 “Successfully logged in” 就行。

> 如果浏览器没弹出，终端里会打印一条网址，手动复制到浏览器打开授权。

### 4.2 创建数据库 D1（存小说正文、角色、记忆）
```bash
npx wrangler d1 create novel_db
```
执行完，终端会打印一段，**重点是这一行**：
```
database_id = "1a2b3c4d-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
```
👉 **把这串 `database_id` 复制下来**，下一步要填进配置文件。

### 4.3 创建 KV（存运行开关、锁、统计）
```bash
npx wrangler kv namespace create KV
```
执行完会打印类似：
```
id = "0f9e8d7c6b5a..."
```
👉 **把这串 `id` 也复制下来。**

> 注意：**Queues（队列）不用创建**。免费计划下系统会自动用定时器写章节，功能一样。

### 4.4 把两个 id 填进配置文件 `wrangler.toml`

用记事本/任意编辑器打开项目根目录下的 **`wrangler.toml`**，找到这两处，把尖括号占位符换成你刚复制的真实值：

找到：
```toml
[[d1_databases]]
binding = "DB"
database_name = "novel_db"
database_id = "<在 dashboard 创建后填入>"
```
改成（用你 4.2 的值）：
```toml
database_id = "1a2b3c4d-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
```

找到：
```toml
[[kv_namespaces]]
binding = "KV"
id = "<在 dashboard 创建后填入>"
```
改成（用你 4.3 的值）：
```toml
id = "0f9e8d7c6b5a..."
```

保存文件。

### 4.5 初始化数据库表结构
```bash
npm run db:init:remote
```
看到一堆 “Executing… ✔” 就成功了（它在远程数据库里建好了所有表）。

✅ **检查点**：D1、KV 的 id 已填入 wrangler.toml 并保存；建表命令成功。

---

## 第 5 步：配置密钥 + 部署

### 5.1 配置 DeepSeek 密钥和后台登录口令
逐条执行，每条会提示你**粘贴一个值再回车**：

```bash
# 粘贴你的 DeepSeek 密钥 sk-...
npx wrangler secret put DEEPSEEK_API_KEY

# 设一个后台登录口令（自己编一串，比如 mySecret123，记住它，登录控制台要用）
npx wrangler secret put ADMIN_TOKEN
```

（可选）想要 Telegram 推送每日报告/报错，再配这两个；不想要就跳过：
```bash
npx wrangler secret put TELEGRAM_BOT_TOKEN
npx wrangler secret put TELEGRAM_CHAT_ID
```

### 5.2 确认模型名
打开 `wrangler.toml`，看这行：
```toml
DEEPSEEK_MODEL = "deepseek-v4-flash"
```
如果 DeepSeek 官网你的可用模型**不叫这个名**（常见是 `deepseek-chat`），把引号里改成正确的名字再保存。不确定就先这样，跑报错了我帮你看。

### 5.3 部署后台 Worker
```bash
npm run deploy
```
成功后终端会打印一条网址，形如：
```
https://novel-engine.你的名字.workers.dev
```
👉 **把这条网址复制保存**，这就是你的后台地址（下面叫它 `<后台网址>`）。

### 5.4 部署网页控制台
```bash
npx wrangler pages deploy console --project-name novel-console
```
第一次它可能问要不要创建项目，确认即可。成功后打印一条网址，形如：
```
https://novel-console.pages.dev
```
👉 这是你的**控制台网址**，用浏览器打开它。

✅ **检查点**：你现在有两条网址——后台 `*.workers.dev` 和控制台 `*.pages.dev`。

---

## 第 6 步：导入大纲，开始自动写

### 6.1 一条命令导入《玄天鼎尊》
把下面命令里的 `<后台网址>` 和 `<你的ADMIN_TOKEN>` 换成你自己的，然后执行：

```bash
python3 tools/parse_outline.py outlines/玄天鼎尊.md --target 1850 \
  --api <后台网址> --token <你的ADMIN_TOKEN> --push
```
（Windows 若 `python3` 不行就用 `python`）

成功会打印：`✅ 已创建书 id=xxxx，并 seed 8 个角色。`

### 6.2 打开控制台开跑
1. 浏览器打开你的控制台网址（`*.pages.dev`）。
2. 右上角两个框：第一个填 `<后台网址>`，第二个填 `<你的ADMIN_TOKEN>`，点 **连接**。
3. “书架”里能看到《玄天鼎尊》。点 **▶ 开始**（或点 **⚡ 立即生成一章** 立刻出一章）。
4. 等 1–2 分钟，切到“章节”标签，刷新，就能看到第 1 章了。点 **📋 一键复制本章**。

> 之后系统每 2 分钟自动写一章，无需你管。想停就点 **⏸ 暂停**。

✅ **大功告成！**

1. **前 3 章完整正文**（控制台一键复制，或浏览器开 `<后台网址>/public/books/<书id>/chapters/1`）。

## 常见报错速查

| 现象 | 原因 / 解法 |
|---|---|
| `node`/`git`/`python` 不是命令 | 对应工具没装好或没加 PATH，重装并重启终端 |
| `wrangler login` 没反应 | 复制终端里打印的网址，手动到浏览器打开授权 |
| 部署报 `D1_ERROR` / 表不存在 | 第 4.5 步建表没跑成功，重跑 `npm run db:init:remote` |
| 部署报和 `queues` 相关的错 | 说明 wrangler.toml 里队列没注释掉；确认用的是最新代码（队列默认已注释） |
| 生成的章节状态一直 error，日志里 `LLM 4xx` | DeepSeek 密钥错了，或 `DEEPSEEK_MODEL` 模型名不对 → 改对后 `npm run deploy` 重新部署 |
| 控制台点连接后空白/报 401 | ADMIN_TOKEN 填错了，和 5.1 设的要一致 |
| 第 1 章迟迟不出 | 等满 2 分钟；或在书架点“⚡ 立即生成一章”；再不行看“日志”页报错发我 |

## 关于花费
- Cloudflare：全程**免费计划**，够用。
- DeepSeek：**唯一花钱的地方**，按生成字数计费，很便宜。建议先充小额，跑十几章看看消耗，心里有数了再放开写。
- 想要更高吞吐（多本书同时高速写）再考虑开 Cloudflare 付费计划启用 Queues（教程在 `wrangler.toml` 注释里），**非必需**。

## 改完代码后如何更新线上
以后我改了代码、你 `git pull` 之后，只要重新跑：
```bash
npm run deploy                                   # 更新后台
npx wrangler pages deploy console --project-name novel-console   # 更新控制台
```
