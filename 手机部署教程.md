# 纯手机部署教程（Cloudflare 网页一键部署版）

> 适用：你已经在 Cloudflare 网站上「连接 GitHub 一键部署」出了后端(Worker)和控制台(Pages)，但点连接没反应。
> 原因：**数据库还没创建并绑定**（`wrangler.toml` 里还是占位符），密钥也还没配。下面全程在手机浏览器里点,不需要电脑、不需要命令行。

你现在的网址（举例，换成你自己的）：
- 后端：`https://jianglong.hejianglong39-21c.workers.dev`
- 控制台：你的 `xxx.pages.dev`

还差 4 件事：① 建数据库 D1 ② 建存储 KV ③ 配 2 个密钥 ④ 把 ①② 的 id 填进配置并重新部署。**建表已自动化，不用你手动跑 SQL。**

---

## A. 创建数据库 D1（拿到 database_id）

1. 手机浏览器打开 https://dash.cloudflare.com/ 登录。
2. 左侧菜单找 **存储和数据库 / Storage & Databases**（旧版在 **Workers & Pages** 里）→ 点 **D1 SQL Database**。
3. 点 **创建 / Create** → 名字填 **`novel_db`** → 创建。
4. 创建后进入该数据库页面，找到 **Database ID**（一串像 `1a2b3c4d-...` 的码）→ **复制**，先存到手机备忘录。

## B. 创建 KV 存储（拿到 id）

1. 同样在 **Workers & Pages**（或 Storage）里找 **KV**。
2. 点 **创建命名空间 / Create namespace** → 名字填 **`KV`** → 创建。
3. 创建后会显示一串 **ID** → **复制**，存备忘录。

## C. 配置 2 个密钥（DeepSeek 密钥 + 登录口令）

1. 在 **Workers & Pages** 列表里点开你的后端 **jianglong**（就是那个 Worker）。
2. 进 **设置 / Settings** → **变量和机密 / Variables and Bindings**。
3. 点 **添加 / Add**，类型选 **机密 / Secret**：
   - 名称：`DEEPSEEK_API_KEY`　值：你的 DeepSeek 密钥 `sk-...`　→ 保存
4. 再点 **添加** 一个 **Secret**：
   - 名称：`ADMIN_TOKEN`　值：自己编一串口令（如 `wo De MiMa 8888`，**记住它**，登录控制台要用）→ 保存

> 这两个保存后会一直生效，不会被覆盖。

## D. 填 id + 重新部署（在 GitHub 手机网页改一个文件）

这一步把 A、B 拿到的两个 id 填进配置文件，保存后 Cloudflare 会**自动重新部署**。

1. 手机浏览器打开你的仓库：`https://github.com/520hejiang/jianglong`
2. **切换到你部署用的分支**。怎么知道是哪个分支？回 Cloudflare → 点开 Worker **jianglong** → **设置/Settings → 构建/Builds** 里能看到 “Branch”。一般是 `claude/xianxia-novel-generator-i0essg`。在 GitHub 页面顶部那个分支下拉框选它。
3. 找到并打开 **`wrangler.toml`** 文件 → 点右上角 **铅笔图标** 编辑。
4. 找到这两行（大约第 18、24 行）：
   ```
   database_id = "<在 dashboard 创建后填入>"
   ```
   ```
   id = "<在 dashboard 创建后填入>"
   ```
   把引号里的占位符分别换成你 A、B 复制的真实 id，例如：
   ```
   database_id = "1a2b3c4d-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
   ```
   ```
   id = "0f9e8d7c6b5a4321..."
   ```
5. **顺便确认模型名**（同一个文件里）：
   ```
   DEEPSEEK_MODEL = "deepseek-v4-flash"
   ```
   如果 DeepSeek 官网你能用的模型不叫这个（很常见是 `deepseek-chat`），就把引号里改成对的。
6. 拉到页面底部，点 **Commit changes / 提交更改**（提交到当前分支即可）。

提交后，Cloudflare 会在 1–3 分钟内自动重新部署（用了你刚填的数据库）。

> 想看进度：Cloudflare → Worker **jianglong** → **构建/Builds**，能看到最新一次部署在跑。

## E. 自检（最关键的一步，确认全配好了）

等部署完（约 2 分钟），手机浏览器打开：
```
https://jianglong.hejianglong39-21c.workers.dev/health
```
（换成你自己的后端网址 + `/health`）

你会看到一段信息，**逐项都该是 ok / 已配置**：
```json
{
  "数据库D1": "ok",
  "KV存储": "ok",
  "模型": "deepseek-v4-flash",
  "DeepSeek密钥": "已配置",
  "登录口令": "已配置"
}
```
- 哪一项显示 **FAIL**，就回去补对应的步骤（D1 FAIL→A/D 没填对；密钥 FAIL→C 没配）。
- 全部 ok 就成功了，继续下一步。

## F. 打开控制台连接

1. 手机浏览器打开你的控制台 `xxx.pages.dev`，能看到 “📜 修仙小说生成控制台”。
2. 右上角两个框：
   - 第一个填后端网址：`https://jianglong.hejianglong39-21c.workers.dev`
   - 第二个填你 C 步设的 `ADMIN_TOKEN`
3. 点 **连接**。现在会有明确提示了：
   - 弹「✅ 连接成功」→ 成了。
   - 弹具体错误 → 按提示修（多半是 token 填错或某项 FAIL）。

## G. 导入《玄天鼎尊》大纲（手机一键，无需电脑）

1. 手机打开仓库里的成品文件：
   `https://github.com/520hejiang/jianglong/blob/claude/xianxia-novel-generator-i0essg/outlines/玄天鼎尊.book.json`
   （在你的部署分支下）
2. 点右上角 **Raw / 原始**，进入纯文本页面 → **全选 → 复制**全部内容。
3. 回控制台「书架」→ 点 **📥 导入大纲JSON** → 把刚才复制的内容**粘贴**进去 → 点 **创建**。
4. 弹「✅ 已创建《玄天鼎尊》并导入 8 个角色」就成了。

## H. 开始自动写

1. 书架里出现《玄天鼎尊》→ 点 **▶ 开始**（或点 **⚡ 立即生成一章** 立刻出一章）。
2. 等 1–2 分钟，切到「章节」标签刷新，第 1 章就出来了，可一键复制。
3. 之后系统每 2 分钟自动写一章，不用管。想停点 **⏸ 暂停**。

✅ **全部完成！**

## 常见问题（手机版）

| 现象 | 解法 |
|---|---|
| `/health` 里 `数据库D1: FAIL` | A 步没建 D1，或 D 步 database_id 没填对/分支选错。回去核对，重新提交触发部署 |
| `/health` 里 `DeepSeek密钥: FAIL` | C 步没加 `DEEPSEEK_API_KEY` 机密 |
| 控制台点连接弹「连不上后台」 | 后端网址填错。要以 `.workers.dev` 结尾、不带最后的斜杠 |
| 连接弹「鉴权失败」 | `ADMIN_TOKEN` 和 C 步设的不一致 |
| 改了 wrangler.toml 但没生效 | 确认改的是**部署用的那个分支**；去 Builds 看有没有触发新部署 |
| 章节状态变 error，日志里 `LLM 4xx` | 模型名不对(D步改 `DEEPSEEK_MODEL`)，或 DeepSeek 余额不足 |
| 找不到 D1/KV 菜单 | Cloudflare 新版在「Storage & Databases」，旧版在「Workers & Pages」里，多翻翻 |
